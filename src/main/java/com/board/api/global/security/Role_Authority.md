이 메서드는 한 줄짜리 코드지만, **자바의 제네릭(Generics), 자료구조(Collections), 그리고 스프링 시큐리티의 객체지향 설계**가 모두 압축되어 있는 아주 밀도 높은 코드입니다.

이 한 줄이 어떻게 구성되어 있는지 **1. 반환 타입(껍데기)**과 **2. 내부 구현(알맹이)**으로 나누어 아주 자세히, 논리적으로 분해해 드릴게요.

---

### 1부: 메서드의 선언부 (반환 타입 파헤치기)

```java
public Collection<? extends GrantedAuthority> getAuthorities()

```

이 부분은 스프링 시큐리티가 **"이 유저가 가진 권한(들)을 내놔!"**라고 요구하는 공식 규격입니다.

#### ① `Collection` (컬렉션)

* **의미:** 자바에서 여러 개의 데이터를 담을 수 있는 **'바구니'**의 최상위 인터페이스입니다. (List, Set 등의 부모)
* **왜 바구니를 쓸까요?** 어떤 유저는 "일반 유저" 권한만 가질 수도 있지만, 어떤 유저는 "일반 유저 + 게시판 관리자 + 전체 관리자"처럼 **여러 개의 권한을 동시에 가질 수도 있기 때문**입니다. 그래서 무조건 바구니(Collection)에 담아서 달라고 요구하는 것입니다.

#### ② `GrantedAuthority` (부여된 권한)

* **의미:** 스프링 시큐리티에서 **'권한'을 나타내는 공식 인터페이스**입니다. 시큐리티는 단순한 `String`("ROLE_USER")을 권한으로 인정하지 않고, 반드시 이 규격을 갖춘 '객체'를 요구합니다.

#### ③ `<? extends ...>` (제네릭 와일드카드)

* **의미:** "이 바구니(`Collection`) 안에는 `GrantedAuthority` 인터페이스를 구현한(상속받은) 객체라면 **그 어떤 종류든 다 담을 수 있다**"는 자바 문법입니다.
* 개발자가 나중에 시큐리티가 제공하는 기본 권한 객체 말고, 입맛대로 커스텀한 권한 객체를 만들어서 바구니에 담아도 에러가 나지 않도록 유연하게 열어둔 것입니다.

### 2부: 내부 구현 로직 (알맹이 만들어서 바구니에 담기)

이제 시큐리티의 요구사항(규격)을 알았으니, 우리가 가진 데이터(`member.getRole()`)를 저 규격에 맞게 변환해서 돌려줘야 합니다.

```java
return Collections.singletonList(new SimpleGrantedAuthority(member.getRole().getKey()));

```

이 코드는 **안쪽에서 바깥쪽으로** 해석해야 합니다.

#### ① 가장 안쪽: `member.getRole().getKey()`

* 앞서 우리가 만든 `MemberRole` enum 객체(`USER` 또는 `ADMIN`)에서 `.getKey()`를 호출합니다.
* **결과물:** `"ROLE_USER"` 또는 `"ROLE_ADMIN"` 이라는 단순한 **문자열(String)**이 튀어나옵니다.

#### ② 중간 포장: `new SimpleGrantedAuthority(...)`

* 시큐리티는 단순한 문자열을 취급하지 않는다고 했죠? 그래서 시큐리티가 제공하는 가장 기본적이고 심플한 권한 객체인 **`SimpleGrantedAuthority`**를 생성(new)합니다.
* 아까 뽑아낸 `"ROLE_USER"` 문자열을 이 객체의 생성자에 쏙 집어넣습니다.
* **결과물:** 단순 문자열이 **'시큐리티 공식 권한 인증 마크를 단 객체'**로 변신했습니다.

#### ③ 최종 포장: `Collections.singletonList(...)`

* 이제 이 객체를 바구니(`Collection`)에 담아서 리턴해야 합니다.
* 보통은 `List<GrantedAuthority> list = new ArrayList<>(); list.add(...);` 처럼 만들지만, 현재 우리 서비스는 **한 유저당 1개의 권한(USER 또는 ADMIN)만** 가지고 있습니다.
* **`singletonList`의 마법:** 자바의 `Collections` 유틸리티가 제공하는 이 메서드는 **"딱 1개의 아이템만 들어가는 아주 가볍고 빠른 리스트"**를 만들어줍니다.
* 빈 공간을 미리 만들지 않기 때문에 메모리를 아낄 수 있고, 수정이 불가능한 '불변(Immutable)' 리스트를 반환하므로 보안상으로도 훨씬 안전합니다.

### 💡 최종 요약 비유

이 한 줄의 코드는 현실 세계의 이런 과정과 똑같습니다.

1. `member.getRole().getKey()` ➡️ 내 서랍에서 **"관리자"라고 적힌 종이 조각**을 꺼낸다.
2. `new SimpleGrantedAuthority(...)` ➡️ 그 종이 조각을 **'회사 공식 신분증 케이스'**에 끼워 넣는다.
3. `Collections.singletonList(...)` ➡️ 그 신분증 1개만 딱 들어가는 **'전용 제출용 봉투'**에 담는다.
4. `return` ➡️ 봉투를 시큐리티 보안 요원에게 건네준다.
